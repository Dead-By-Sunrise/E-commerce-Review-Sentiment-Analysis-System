# 电商产品评论情感分析系统

## 主要研究内容

本毕业设计旨在构建一个完整的端到端系统，能够自动从互联网（如电商平台）爬取产品评论，利用机器学习模型对评论进行情感倾向性分析，并通过丰富的图表和交互界面将分析结果直观地展示给用户。主要研究内容包括：

### 1. 数据采集与预处理模块

- **数据采集**：编写网络爬虫程序，从目标电商网站（如京东、天猫）定向爬取指定产品的评论数据，包括评论内容、评分、用户昵称、评论时间等。
- **数据清洗与预处理**：对爬取到的原始评论文本进行清洗，包括去除无关符号、纠正错别字、处理缩写等。进行中文分词（使用Jieba等工具）、去除停用词等自然语言处理预处理工作。

### 2. 情感分析模型构建与训练

- **特征工程**：将文本评论转化为机器学习模型可处理的数值特征。方法包括：
  - 传统方法：词袋模型、TF-IDF
  - 深度学习方法：词嵌入，如Word2Vec、GloVe或上下文相关的Embedding
- **模型选型与实现**：
  - 传统机器学习模型：实现并比较如朴素贝叶斯、支持向量机、逻辑回归等分类器
  - 深度学习模型：实现更复杂的模型，如TextCNN、BiLSTM、或微调预训练模型以获取更优性能
- **模型训练与评估**：使用已标注的评论数据集（可自行标注或使用公开数据集）对模型进行训练和调优。采用准确率、精确率、召回率、F1分数等指标评估模型性能。

### 3. 系统设计与实现

- **系统架构**：设计前后端分离的系统架构
  - 后端：使用Python（Flask/Django）或Go框架，提供数据爬取、情感分析、数据查询等API接口
  - 前端：使用Vue.js、React等框架开发交互式可视化界面
- **核心功能模块**：
  - 评论管理模块：支持输入产品链接进行爬取，或手动导入评论数据
  - 情感分析模块：调用训练好的模型对评论进行批量或实时分析
  - 可视化展示模块：将分析结果以多种图表形式展示

### 4. 可视化展示模块

- 整体情感分布：使用饼图或环形图展示积极、消极、中性评论的占比
- 情感趋势分析：使用折线图展示产品在不同时间段的评价变化趋势
- 观点摘要与关键词提取：使用词云图突出显示评论中出现的高频情感词和产品特征词
- 细粒度方面情感分析：不仅分析整体情感，还能分析针对产品具体方面的情感倾向，并用柱状图等进行对比

## 目标和要求

### 总体目标

成功开发一个集数据获取、智能分析、结果可视化于一体的产品评论分析系统原型，并能通过案例演示其完整工作流程和有效性。

### 具体目标

1. **功能目标**：实现评论爬取、数据清洗、情感分析、结果可视化等核心功能闭环
2. **性能目标**：情感分析模型在测试集上达到可接受的准确率；系统界面响应流畅
3. **质量目标**：系统架构清晰，代码规范，具备良好的可扩展性和可维护性

### 基本要求

1. 系统必须是完整的、可运行的
2. 情感分析功能必须基于自行训练的机器学习模型实现
3. 可视化界面必须直观易懂，至少包含三种不同类型的统计图表

## 特色

1. **端到端的系统集成**：将数据爬取、文本处理、机器学习建模、Web系统开发和数据可视化等多个技术环节有机整合，体现了软件工程的系统性思维
2. **多维度可视化分析**：不仅展示简单的情感比例，还从时间趋势、关键词、产品维度等多个角度进行深度挖掘和可视化，提供更全面的决策支持
3. **引入方面级情感分析**：实现更细粒度的方面情感分析

## 成果形式

1. **可运行系统一套**：包含完整源代码、训练好的模型文件、数据库文件、项目依赖说明和部署文档

3. **相关技术文档**

## 运行说明

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 清洗数据+商品分类

```bash
python3 data_clean_store.py \
  --source-dir goods \
  --clean-out-dir goods_cleaned \
  --export-csv
```

### 3. 运行分析流水线（可选，用于刷新模型与方面分析结果）

```bash
python3 aspect_sentiment_pipeline.py \
  --input-csv goods_cleaned/datasets/all_comments_clean.csv \
  --output-dir analysis_outputs_full \
  --models lexicon,textcnn,bilstm,bert
```

### 4. 启动 Flask 服务

```bash
python3 app.py
```

### 5. 浏览器访问

| 系统 | 地址 |
|------|------|
| 开发者模型看板 | `http://127.0.0.1:5000/dev` |
| 用户评论分析大屏 | `http://127.0.0.1:5000/user` |

### 主要 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/models/comparison` | 模型指标对比 |
| GET | `/api/dev/model-reports` | 训练分类报告原文 |
| GET | `/api/dev/manifest` | 本次训练配置与最佳模型 |
| GET | `/api/dev/label-distribution` | 标签分布诊断 |
| GET | `/api/sentiment/distribution` | 情感分布（支持 `goods_id` 或 `category_name`） |
| GET | `/api/sentiment/trend` | 时间趋势 |
| GET | `/api/aspects/summary` | 方面情感分布 |
| GET | `/api/aspects/compare` | 同类商品多维雷达对比 |
| GET | `/api/keywords` | 关键词高频统计 |
| GET | `/api/comments` | 评论分页明细 |
| GET | `/api/user/goods-compare` | 多商品核心指标对比 |
| POST | `/api/reload` | 重新加载 CSV 数据 |
